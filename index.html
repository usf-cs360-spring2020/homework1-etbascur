Description

For this homework I got data from the Data SF website and created Three visualizations with Tableau and then attempted to recreate to of these visualizations in D3.
The Tableau and D3 visualizations can be reached by the links at the top of the page, where you can find comparisons of the Tableau/D3 visualizations and the standalone Tableau visualization as well. This is my first attempt at data manipulation and visualization with D3.

Data

The data set was taken from Data SF and shows the Air Traffic Passenger Statistics for the San Francisco international airport.
It collects data by monthly period from July 2005 to September 2019 (as of this date in February 2020)

All data are under Public Domain Dedication and License (PDDL).

Data SF license information

The data itself is divided into 12 columns, according to the Data Dictionary there are 11 attribute fields and 1 fact field, which is Passenger Count

The columns are as follows:

  1. Activity Period: The year and month at which passenger activity took place. It's a numeric type with a range from 200507-201909.
  2. Operating Airline: Airline name for the operator of aircraft with passenger activity. It is a text type, with a range of Airline names.
  3. Operating Airline IATA Code:The International Air Transport Association (IATA) two-letter designation for the Operating Airline. It is a text type.
  4. Published Airline: Airline name that issues the ticket and books revenue for passenger activity. It is a text type
  5. Published Airline IATA Code: The International Air Transport association (IATA) two-letter designation for the Published Airline. It is a text type
  6. GEO Summary: Provides a more detailed breakdown of the GEO Summary field to designate the region in the world where activity in relation to SFO arrived from or departed to without stops. It is a text type.
  7. GEO Region:Provides a more detailed breakdown of the GEO Summary field to designate the region in the world where activity in relation to SFO arrived from or departed to without stops. It is a text type.
  8. Activity Type Code: A description of the physical actiona passenger took in relation to a flight, which includes boarding a flight (“enplanements”), getting off a flight (“deplanements”) and transiting to another location (“intransit”). It is a text type.
  9. Price Category Code: A categorization of whether a Published Airline is a low-cost carrier or not a low-cost carrier. It is a text type.
  10. Terminal: The airport terminal designations at SFO where passenger activity took place.It is a text type.
  11. Boarding Area: The airport boarding area designations at SFO where passenger activity took place. It is a text type
  12. Passenger Count: The number of monthly passengers associated with the above attribute fields.It is a numeric field.

Wrangling

The wrangling for this dataset was mostly done in the Data SF website through their own data manipulation tool, and then exported into Tableau and Atom as CSV files where they were slightly further altered for the data visualizations.

The datasets used for the D3 visualizations were broken down into 3 smaller sets:

  1. For the standalone Tableau visualization, the dataset uses the Activity period, which was changed into a date Type in Tableau, and changed to measure by month on the columns for the year 2019.
    The row is the sum of the Passenger Count associated with each month. It also uses the Geo Summary to divide the information between Domestic and International by color.
  2. For the bar chart, I used the Operating Airline, Passenger Count sum and Geo Summary. I used the Passenger count on the row and filtered it so it showed only the airlines that had over 2 million on the passenger count. I also filtered the Geo Summary to consider only the Domestic flights.
    for the columns I used the Operating Airlines that filled that criteria.
